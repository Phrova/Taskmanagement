Task Business Rules

Task Status Flow:
- New task starts with status = TODO
- When user starts working, change status to IN_PROGRESS
- When work is finished, change status to DONE
- Cannot change from DONE back to TODO or IN_PROGRESS
- Can change from IN_PROGRESS back to TODO if not done yet

Task Assignment:
- One task can only be assigned to one user
- One user can have many tasks
- When creating a task, user can be empty meaning no one assigned yet
- When task is DONE, cannot change the assigned user anymore

Task Fields:
- Title: required, cannot be empty
- Description: optional, can be null
- Status: required, default is TODO
- Create date: auto set to today, cannot be null
- Due date: optional, but if given must be same day or after create date
- User: optional, can be null
- Project: optional, can be null

Task - User Relationship:
- A task belongs to one user if assigned
- If user is deleted, task still stays but user_id becomes null
- Cannot assign task to a user that does not exist
- User must belong to the same project as the task before getting assigned

Task - Project Relationship:
- A task can belong to one project or none
- If task belongs to a project, that project must exist
- A project can have many tasks

Why these rules:
- To track work progress clearly
- To avoid assigning same task to multiple people at once
- To keep data correct when users or projects get deleted
- To make sure due dates actually make sense
