Review Relationship

1 - User & Task (R: One to Many)
_One user can be assigned multiple tasks
_Tasks can be assigned to only one user,the same task can't be assigned to two multiple Users

Explain: Users can be assigned multiple tasks base on their skills. However to maintain efficiency, a single task
can't be shared by multiple users. If a task is too large, it must be decomposed into smaller tasks, each assigned 
to a single user


2 - User & Project (R: Many to Many)
_One project can have multiple users
_One user can participate in multiple projects

Explain: Based on real-life scenarios, a student can handle multiple personal projects,
participate in several school projects and same with user, a project can have multiple students


3 - Project & Task (R: One to Many)
_One project can have several tasks
_Tasks can belonging to multiple projects

Explain: Each project has a different purpose, structure, and plan. So we cannot apply the same tasks to several projects,
it wouldn't be an efficient method and risks ruining the project
